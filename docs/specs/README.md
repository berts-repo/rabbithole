# Specs

This directory contains the original prompt/spec source material and product
surface contracts used to shape the app. Start with `index.md` for the spec
map.

Prefer current code and `docs/reference/` when these notes disagree with the
present system.

## Agent Orientation

For general project context, start with `../../CONTEXT.md`. For current active
implementation work, follow `../work/ACTIVE.md`.

For spec reconstruction or UI/product intent, read in this order:

1. `../../CONTEXT.md`
2. `index.md`
3. The relevant spec file under this directory
4. The relevant current-truth doc under `../reference/`
